# Agent 执行器架构分析

## 概述

Agent 执行器（Agent Runner）是 AstrBot v4.7.0 引入的核心架构抽象，负责"思考 + 做事"，与只负责"说话"的 Chat Provider 形成职责分离。AstrBot 目前支持五种执行器：内置、Dify、Coze、阿里云百炼、DeerFlow。

## 核心设计问题

### 为什么不把 Dify/Coze/百炼当作普通 Chat Provider？

早期这些平台作为普通 Chat Provider 集成，但它们自带 Agent 能力，与传统"单轮文本补全" Provider 有本质差异：

| 维度 | Chat Provider | Agent Runner |
|------|--------------|--------------|
| 输入 | prompt + 历史 + 工具列表 | 用户意图 + 上下文 + 环境状态 |
| 输出 | 文本 / 工具调用 | 多轮循环：感知→规划→执行→观察→再规划 |
| 本质 | 单轮补全接口 | 策略驱动的动作循环 |
| 代表 | OpenAI、Claude | Dify、Coze、百炼应用 |

强行将 Agent Runner 当作 Chat Provider 会导致功能冲突：外部平台已在内部实现了 Agent 循环，若再套一层 AstrBot 内置执行器，会产生双重循环问题。

## 架构分层

```
┌─────────────────────────────────────────────────────┐
│                  AstrBot 上层应用                     │
├─────────────────────────────────────────────────────┤
│  Agent Runner（思考 + 做事）                          │
│  ┌─────────┬─────────┬─────────┬─────────┬────────┐ │
│  │ 内置Runner │ Dify   │ Coze    │ 百炼    │ DeerFlow│ │
│  └────┬────┴────┬────┴────┬────┴────┬────┴────┬────┘ │
│       └──────────────┬──────────────────────────┘    │
│                      ▼                               │
│  Chat Provider（说话：文本补全）                       │
│  ┌─────────┬─────────┬─────────┬─────────┐         │
│  │OpenAI  │ Claude  │ Dify    │ 百炼    │ ...    │
│  └─────────┴─────────┴─────────┴─────────┘         │
└─────────────────────────────────────────────────────┘
```

## Agent Runner 执行循环

```
用户意图 → 感知(Observe) → 规划(Plan) → 执行(Act) → 观察(Observe) → ... → 结束
```

1. **感知（Observe）**：读取环境状态、工具返回结果、对话历史
2. **规划（Plan）**：基于策略/模型决定下一步动作
3. **执行（Act）**：调用工具、操作环境
4. **循环**：重复直到任务完成或超时

## 内置执行器能力

内置执行器配合 AstrBot 生态可实现：

- **MCP 集成**：通过 MCP 协议调用外部工具
- **知识库**：向量检索增强回答
- **网页搜索**：实时信息获取
- **代码执行**：Local（风险）或 Sandbox（隔离）环境

## 执行环境

| 环境 | 说明 | 风险 |
|------|------|------|
| Local | Agent 在 AstrBot 运行环境中执行 | 高风险：任意代码执行权限 |
| Sandbox | 隔离沙盒环境 | 低风险：权限受限 |

Local 模式下仅管理员请求可真正执行，普通用户会被拒绝。

## Runner 类型一览

| Runner | 来源 | 特点 |
|--------|------|------|
| 内置 Runner | AstrBot 原生 | 配合 MCP/知识库/搜索 |
| Dify Runner | Dify 平台 | Dify 内置 Agent 循环 |
| Coze Runner | Coze 平台 | Coze 内置 Agent 循环 |
| 百炼 Runner | 阿里云百炼 | 百炼应用内置 Agent 循环 |
| DeerFlow Runner | DeerFlow | DeerFlow 内置 Agent 循环 |

## 设计价值

- **职责清晰**：Provider 管补全，Runner 管执行
- **扩展灵活**：新增 Runner 只需实现接口，无需改动核心
- **生态兼容**：内置 Runner 享有 MCP、知识库等 AstrBot 特有能力；外部 Runner 则复用外部平台生态
